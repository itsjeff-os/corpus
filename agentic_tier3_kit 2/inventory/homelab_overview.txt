Conventions & Protocols

Ownership / workspace
	•	/srv is owned by sysadmin:sysadmin and treated as the primary workspace for services.

New Stack Protocol (NSP)

When adding a new long-lived stack:
	1.	Location
	•	Apps: /srv/apps/<name>/
	•	Infra/glue: /srv/infra/<name>/
	2.	Scaffold
	•	Create the folder under /srv/....
	•	Add docker-compose.yml with service name, image/build, ports, volumes.
	•	Add README.md with purpose, start/stop commands, ports/URLs, and dependencies.
	3.	Aliases (in ~/.bash_aliases)
	•	cd-<name> → cd /srv/.../<name>
	•	<name>-up → cd /srv/.../<name> && docker compose up -d
	•	<name>-down → cd /srv/.../<name> && docker compose down
	•	<name>-logs → docker logs -f <container_name>
	4.	Docs / CLI
	•	Add a row to HOMELAB.md under Stacks.
	•	Add core commands to the Homelab CLI Canvas (and optionally HOMELAB_CLI.md).
